RequireJS 模块化加载 与 uglify 文件压缩

Posted on 2016-04-19   |   In JavaScript / JQuery

关于 RequireJs 它可以用来管理页面中使用的 JS 库之间的依赖关系，同时它也可以让我们以模块化的形式组织 JS 代码。而 uglify 就是将 JS 文件自动化压缩，可以设置混淆变量名，合并 JS 文件等。

安装 RequireJS 与 uglify

RequireJS ：bower install requirejs

uglify ：npm install grunt-contrib-uglify --save-dev

RequireJS 的使用

在适当的位置引用，其中 require.js 是我们通过 Bower 下载下来的 RequireJS，而 data-main 中定义的则是 RequireJS 的配置文件，需要我们手动新建和修改（config.js）,在引用时不需要添加.js后缀。

RequireJS 的 config 配置文件

require.config({
  //paths 模块声明（不需要添加.js后缀）
  paths: {
    'test' : 'test',  //test 是与 config.js 同级目录下的 test.js 文件
    'jquery': '../../assets/jquery/js/jquery.min'
  },
  shim: {
    //处理一些没有遵守 AMD 规范的 js 库
    'underscore':{
      exports: '_'  //exports值（输出的变量名），表明这个模块外部调用时的名称
    },
    'backbone': {
      deps: ['underscore', 'jquery'], //deps数组，表明该模块的依赖性
      exports: 'backbone'
    }
  }
});

//加载相应 JS 文件
require(['test','jquery'],function(t) {
  t.hello();  //执行回调函数
});

test.js 文件中 RequireJs 模块式的写法

define([
  'jquery'  //引用 jQuery
  ], function($) {
  return {
    hello: function(){
      console.log('Hello World');
      $('.container').hide();
    }
  };
});

uglify 文件压缩的使用

通过 Gruntfile.js 配置文件来添加 uglify 文件压缩任务

module.exports = function (grunt) {
  // 项目配置
  grunt.initConfig({
    //压缩文件
    uglify:{
      // 加注释
      options:{
        banner:"/* <%= pkg.name="" %=""> <%= pkg.version="" %="">*/ \n"
      },
      buildOne:{
        files:{
          //将 config.js 压缩成 config.min.js
          'dist/js/config.min.js' : 'src/js/config.js'
        }
      },
      buildAll:{
        //将某个文件夹下的所有JS文件按结构压缩到指定的文件夹下
        files:[{
          expand: true,
          cwd:'src/js',   //JS目录下
          src:'**/*.js',  //所有JS文件
          dest:'dist/js'  //压缩到某个目录
        }]
      }
    }
  });
  //加载
  grunt.loadNpmTasks('grunt-contrib-uglify');  //文件压缩

  //执行任务
  grunt.registerTask('buOne', ['uglify:buildOne']);  //grunt buOnt
  grunt.registerTask('buAll', ['uglify:buildAll']);  //grunt buAll
};
